SOLVED PROBLEMS (kept for reference — canonical docs are in smgp.asm source comments)
===============================================================================
- Slope data encoding: fully documented in Update_slope_data / Integrate_curveslope headers.
  Visual slope at Visual_slope_data ($00FF7300), physical slope at Physical_slope_data ($00FF8300).
  3/4-byte RLE records, delta encoding, BMI sentinel ($80+). See TRACK-002.

- Sign data encoding: fully documented in Parse_sign_data / Parse_tileset_for_signs headers.
  4-byte records (distance.w, count.b, sign_id.b), $FFFF sentinel resets each lap.
  Sign constants in constants.asm ($FFFF9240–$FFFF925C range). See TRACK-003.

- Track_data record layout: fully documented in Load_track_data header ($48 bytes/entry,
  all 8 pointer fields + 12 scalar fields named; all 19 tracks labeled). See TRACK-001.

- Curve data encoding: documented in smgp.asm near Curve_data ($00FF5B00).
  Bytes: $00=straight, $01-$2F=left (extreme→soft), $41-$6F=right, $80+=lap wrap.

- Player state struct at Player_obj ($FFFFAE00): fully documented and named.
  Key fields: Horizontal_position, Player_distance, Player_place_score.
  See Car_obj_* field constants in constants.asm.

- AI car array at Ai_car_array ($FFFFB080): 16×$40-byte entries.
  Entry 0 = Rival_car_obj. Same Car_obj_* field offsets apply.

- All loc_ label definitions eliminated (0 loc_ definitions remain in smgp.asm).
  ~3099 loc_XXXX preservation comments remain as cross-reference anchors.

CURVE DATA DECOMPRESSED FORMAT
================================================================================
Stored in Curve_data ($00FF5B00), read by Update_slope_data / Integrate_curveslope.
One byte per step forward. Ends with $FF ($80+).
Code reads 40 bytes ahead; ca 26 steps are rendered.

Byte values:
* $00       = Straight
* $01-$2F   = Left turn, from extreme (01) to soft (2F)
* $30-$3F   = Game freezes (invalid)
* $41-$6F   = Right turn, from extreme (41) to soft (6F)
* $70-$7F   = Game freezes (invalid)
* $80-$FF   = Sentinel / wrap: continue reading from track start (new lap)

Lower 6 bits = sharpness (higher = softer). Bit 6 = right turn. Bit 7 = wrap sentinel.

Background horizontal displacement ($00FF6300 = Background_horizontal_displacement):
Encoded together with curve data in ROM as relative delta values.
In memory stored as absolute rotation. Update_slope_data integrates via ADD/SUB each frame.


SLOPE DATA DECOMPRESSED FORMAT (SOLVED — see Update_slope_data header in smgp.asm)
================================================================================
Visual slope:  Visual_slope_data  ($00FF7300) — drives background vertical shift
Physical slope: Physical_slope_data ($00FF8300) — drives RPM drag via Track_phys_slope_value

Byte encoding identical to curve data:
* $00       = Flat
* $01-$2F   = Down slope (extreme → soft)
* $41-$6F   = Up slope (extreme → soft)
* $80+      = Sentinel (BMI branch terminates RLE loop)

ROM encoding: 3/4-byte RLE records (initial displacement byte + per-record count + delta).
Physical slope uses simpler 3-byte RLE. Both terminated by high-byte bit 7 set.
Track_phys_slope_value is used by Update_rpm: negative = uphill drag on engine.
